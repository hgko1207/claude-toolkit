# 스킬 크리에이터

새로운 Claude Code 스킬을 생성합니다.

## 핵심 원칙

- **간결하게**: 컨텍스트 윈도우는 공유 자원. Claude가 이미 아는 것은 쓰지 않음.
- **자유도 조절**: 유연한 작업은 높은 자유도, 정밀한 작업은 낮은 자유도.
- **SKILL.md는 500줄 이하**: 초과 시 references/ 폴더로 분리.

## 스킬 구조

```
.claude/skills/스킬이름/
├── SKILL.md          # 필수: 프론트매터 + 지침
├── scripts/          # 선택: 반복 작업용 스크립트
├── references/       # 선택: 조건부 로딩 문서
└── assets/           # 선택: 템플릿, 리소스
```

## 프론트매터 필드

```yaml
---
name: 스킬이름 # 필수: 소문자+하이픈
description: 언제 사용하는지 # 필수: 자동 트리거 기준
argument-hint: [인자 설명] # 선택: 인자 힌트
disable-model-invocation: true # 선택: 도구 호출 비활성화
---
```

## 생성 워크플로우

1. **요구사항 파악**: 사용자에게 스킬 용도, 트리거 조건, 필요 기능을 확인
2. **범위 결정**: 프로젝트(`.claude/skills/`) 또는 전역(`~/.claude/skills/`)
3. **구조 설계**: SKILL.md만으로 충분한지, scripts/references가 필요한지 판단
4. **작성**: 프론트매터 + 지침을 간결하게 작성
5. **테스트**: `/스킬이름`으로 호출하여 동작 확인

## 좋은 description 예시

```yaml
# 좋음 - 구체적 트리거
description: 빌드 후 커밋하고 Vercel에 배포. "배포해줘" 요청 시 사용.

# 좋음 - 명확한 용도
description: plan.md에 요청사항을 정리. "플랜 작성해줘" 요청 시 사용.

# 나쁨 - 모호함
description: 코드 도움
```

## 작성 지침

- 명령형으로 작성 ("~합니다" 대신 "~하세요")
- 단계별 워크플로우를 번호로 나열
- 규칙/제약은 별도 섹션으로 분리
- 프로젝트 특화 정보는 references/에 분리
