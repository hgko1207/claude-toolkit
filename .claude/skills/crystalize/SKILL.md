# 프롬프트 압축기 (Crystalize)

긴 프롬프트를 의미를 보존하면서 토큰 사용량을 최소화하도록 압축합니다.

## 압축 원칙

### 1. 의도 보존

각 지침의 근본 목적(WHY)은 반드시 유지. 표현만 간결하게.

### 2. 고해상도 토큰화

장황한 설명 → 도메인 전문 용어/핵심 키워드로 대체.

- Before: "사용자가 입력한 데이터가 올바른 형식인지 확인하고 잘못된 경우 에러를 반환"
- After: "입력 검증 → 실패 시 에러 반환"

### 3. 암묵지 활용

LLM이 이미 아는 것은 "적용", "참고", "기준" 등 신호만 전달.

- Before: "SOLID 원칙 중 단일 책임 원칙을 따라서 하나의 클래스는 하나의 책임만..."
- After: "SRP 적용"

### 4. 조건부 확장 구조

`[조건] → [행동]` 패턴으로 구조화.

- Before: "만약 빌드가 실패하면 에러를 확인하고 수정한 다음 다시 빌드하세요"
- After: "빌드 실패 → 에러 수정 → 재빌드"

### 5. 중복 제거

다른 표현으로 반복된 동일 의미를 통합.

## 워크플로우

1. 원본 프롬프트를 받음
2. 각 섹션별로 위 5가지 원칙 적용
3. 압축된 버전 출력
4. 압축 보고서 출력:
   - 압축률 (원본 토큰 → 압축 토큰)
   - 섹션별 압축 방식
   - 정보 손실 가능 영역

## 출력 형식

```
## 압축 결과
(압축된 프롬프트)

## 압축 보고서
- 압축률: XX% (원본 ~N토큰 → 압축 ~N토큰)
- 주요 변환: ...
- 주의: 의미 손실 가능 영역
```
