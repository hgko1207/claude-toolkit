# Claude 정보 노트 작성기 (write-note)

Claude, Claude Code, Cowork 등 Claude 관련 소스를 분석하여 `tips/` 폴더에 마크다운 노트로 정리합니다.

## 실행 프로세스

### Step 1: 소스 분석
- 제공된 URL 또는 내용을 깊이 읽고 핵심을 추출한다
- 단순 요약이 아닌, 배경 지식과 실전 인사이트를 추가한다
- 사실 관계를 확인하고 누락된 맥락을 보완한다

### Step 2: 글 유형 판별
내용의 성격에 따라 유형을 결정한다:

| 유형 | 판별 기준 | 구조 |
|------|-----------|------|
| **팁 모음형** | 여러 팁/트릭을 나열 | 문제 → 해결 → 핵심 |
| **심층 가이드형** | 하나의 주제를 깊이 다룸 | Part 분할 + 테이블 |
| **레퍼런스형** | 명령어/설정/API 정리 | 테이블 + 코드블록 중심 |

### Step 3: 폴더 분류
CLAUDE.md의 폴더 구조를 읽고 적절한 위치를 결정한다.
기존 폴더에 맞지 않으면 새 폴더를 생성한다.

### Step 4: 파일 작성

#### 파일명 규칙
- 소문자 영문, 하이픈(`-`) 구분
- 예: `claude-code-hooks-tips.md`, `claude-opus-4-analysis.md`

#### 필수 요소
모든 글에 반드시 포함:

```markdown
# 제목 (핵심을 한 줄로, 검색 친화적)

> 한 줄 요약: 이 글에서 다루는 핵심 내용을 한 문장으로
```

- **H1 제목**: 명확하고 검색되기 쉬운 제목
- **한 줄 요약**: `>` blockquote로 시작
- **테이블**: 비교/정리 시 적극 활용
- **코드블록**: 설정/명령어는 반드시 코드블록으로
- **출처**: 글 하단에 원본 링크

#### 선택 요소 (해당 시 포함)
- **목차**: H2가 5개 이상인 긴 글
- **핵심 요약**: 3~5줄 핵심 포인트 (긴 글에 권장)
- **작성일**: `**작성일**: YYYY-MM-DD`
- **태그**: `**태그**: #ClaudeCode #팁` (티스토리 발행용)

### Step 5: 글 하단 메타 정보

```markdown
---
**출처**: [원본 제목](URL)
**작성일**: YYYY-MM-DD
**태그**: #관련태그1 #관련태그2
```

## 작성 원칙

1. **합쇼체 통일** — ~입니다, ~합니다, ~있습니다 체를 사용 (해체/해요체 금지)
2. **핵심 먼저** — 결론/요약을 글 상단에 배치 (역피라미드 구조)
3. **쉬운 말** — 전문 용어는 처음 나올 때 간단히 설명
4. **구조화** — 글머리 기호, 표, 번호 목록을 적극 활용
5. **실용적** — "그래서 뭘 할 수 있는데?"에 답하는 내용 포함
6. **적절한 길이** — 핵심에 집중, 불필요한 장황한 설명 제거
7. **출처 명시** — 원본 소스 링크 항상 포함

## 티스토리 블로그 친화

- **제목**: 검색되기 쉬운 명확한 제목
- **소제목(H2, H3)**: 스캔하며 읽을 수 있도록 의미 있는 소제목
- **단락 길이**: 한 단락 3~4문장 이내, 모바일 가독성 고려
- **시각 요소**: 테이블, 코드블록을 글 중간에 배치

## 콘텐츠 범위

Claude 관련 정보만 다룬다:
- Claude 모델 (Opus, Sonnet, Haiku 등)
- Claude Code (CLI, 에이전트, 스킬, MCP, 워크플로우)
- Claude 생태계 (API, SDK, Console, Cowork, Desktop)
- Claude 활용 (프롬프팅, 실전 팁, 사례)

범위 밖의 정보는 사용자에게 확인 후 처리한다.
