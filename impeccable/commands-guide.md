# Impeccable 명령어 가이드

> 20개 명령어로 UI/UX 품질을 체계적으로 관리합니다.

---

## 검사/리뷰

---

### /teach-impeccable — 프로젝트 디자인 컨텍스트 수집

#### 무엇을 하는가

프로젝트의 디자인 시스템, 기술 스택, 브랜드 가이드라인을 분석하여 `.impeccable.md` 파일을 생성합니다.
이후 모든 명령어가 이 컨텍스트를 참조하여 프로젝트에 맞는 판단을 내립니다.

#### 사용법

```
/teach-impeccable
```

프로젝트 루트에서 최초 1회 실행합니다. Claude가 디자인 관련 파일들을 탐색하고 질문한 뒤 `.impeccable.md`를 생성합니다.

#### 꿀팁

- 프로젝트 시작 시 가장 먼저 실행합니다 — 다른 모든 명령어의 기반입니다
- 디자인 시스템이 변경되면 다시 실행하여 컨텍스트를 갱신합니다
- Figma 링크, 브랜드 가이드 문서가 있으면 함께 제공하면 더 정확합니다

### /audit [area] — 기술적 품질 검사

#### 무엇을 하는가

5가지 차원에서 **기술적 품질을 정량 평가**합니다.
각 차원 0~4점, 총 20점 만점입니다. 수정하지 않고 리포트만 생성합니다.

| 차원       | 검사 내용                          |
| ---------- | ---------------------------------- |
| 접근성     | ARIA, 키보드 내비게이션, 색 대비   |
| 성능       | 렌더링, 번들 사이즈, 리소스 로딩   |
| 테마       | 다크모드, 토큰 일관성, 커스텀 속성 |
| 반응형     | 브레이크포인트, 터치 타겟, 뷰포트  |
| 안티패턴   | 하드코딩, 인라인 스타일, 매직 넘버 |

발견된 이슈에는 **P0~P3 심각도**가 태깅됩니다.

#### 사용법

```
/audit                    # 전체 프로젝트 검사
/audit src/components/    # 특정 영역 지정
/audit accessibility      # 특정 차원만 집중 검사
```

#### 꿀팁

- `/audit` → `/normalize` → `/polish` 순서가 전형적인 품질 개선 워크플로우입니다
- P0(치명적) 이슈를 먼저 해결하고 P3(사소한) 순서로 진행합니다
- 배포 전 최소 1회 실행하여 기술 부채를 수치로 파악합니다

### /critique [area] — UX 디자인 리뷰

#### 무엇을 하는가

**UX 관점**에서 디자인을 리뷰합니다. 기술적 품질이 아닌 사용자 경험에 집중합니다.

| 평가 항목    | 설명                                     |
| ------------ | ---------------------------------------- |
| 시각적 위계  | 정보 우선순위가 시각적으로 명확한가      |
| 정보 설계    | 구조와 내비게이션이 직관적인가           |
| 감성적 공감  | 브랜드 톤과 사용자 감정에 부합하는가     |
| 인지 부하    | 한 화면에서 처리해야 할 정보가 적절한가  |

**페르소나 기반 테스트**를 포함합니다 — 다양한 사용자 유형이 이 UI를 어떻게 경험할지 시뮬레이션합니다.

#### 사용법

```
/critique                  # 전체 리뷰
/critique checkout         # 결제 플로우 집중 리뷰
/critique landing-page     # 랜딩 페이지 리뷰
```

#### 꿀팁

- `/audit`는 기술적 검사, `/critique`는 디자인 감각 — 둘 다 실행하면 완전합니다
- "모바일 초보 사용자 관점에서 봐줘"처럼 페르소나를 직접 지정할 수 있습니다
- 리뷰 결과를 바탕으로 `/clarify`, `/distill` 등 수정 명령어를 이어서 사용합니다

## 수정/정리

---

### /normalize [feature] — 디자인 시스템 표준 정렬

#### 무엇을 하는가

디자인 시스템 표준에 맞게 코드를 **정렬**합니다.
토큰, 스페이싱, 컴포넌트 패턴을 일관되게 통일합니다.

#### 사용법

```
/normalize                 # 전체 정규화
/normalize sidebar         # 사이드바 영역 정규화
/normalize src/components/ # 특정 경로 지정
```

#### 꿀팁

- `.impeccable.md`에 디자인 토큰이 정의되어 있을수록 정확합니다
- `/audit`에서 "안티패턴" 점수가 낮으면 `/normalize`가 필요하다는 신호입니다
- 하드코딩된 `px` 값, 인라인 색상 등을 디자인 토큰으로 변환합니다

---

### /polish [target] — 출시 전 최종 패스

#### 무엇을 하는가

출시 직전 **마이크로 디테일을 수정**합니다.
정렬 미세 조정, 간격 일관성, 시각적 글리치 제거에 집중합니다.

#### 사용법

```
/polish                    # 전체 최종 패스
/polish header             # 헤더 영역만
/polish settings-page      # 설정 페이지만
```

#### 꿀팁

- `/normalize` 이후에 실행합니다 — 큰 정리 후 세밀한 마무리 순서입니다
- 1px 어긋남, 미세한 색상 불일치 같은 "느낌적 느낌"을 잡아줍니다
- 출시 전 마지막 단계로 사용합니다: `/audit` → `/normalize` → `/polish`

---

### /distill [target] — 복잡성 제거

#### 무엇을 하는가

**본질만 남기고 불필요한 복잡성을 제거**합니다.
과도한 장식, 중복된 요소, 사용하지 않는 상태를 정리하여 깔끔하고 집중된 UI를 만듭니다.

#### 사용법

```
/distill                   # 전체 단순화
/distill dashboard         # 대시보드 단순화
/distill modal             # 모달 컴포넌트 단순화
```

#### 꿀팁

- `/critique`에서 "인지 부하가 높다"는 피드백이 나오면 `/distill`을 실행합니다
- 기능 제거가 아닌 **표현의 단순화**입니다 — 기능은 유지됩니다
- `/bolder`와 반대 방향입니다 — 너무 많은 요소가 문제일 때 사용합니다

---

### /clarify [target] — UX 카피 개선

#### 무엇을 하는가

에러 메시지, 버튼 라벨, 안내문, 플레이스홀더 등 **UX 텍스트를 명확하게 개선**합니다.
사용자가 "이게 무슨 뜻이지?"라고 생각하는 순간을 제거합니다.

#### 사용법

```
/clarify                   # 전체 카피 개선
/clarify forms             # 폼 관련 텍스트 개선
/clarify error-messages    # 에러 메시지 집중 개선
```

#### 꿀팁

- 개발자가 작성한 에러 메시지는 대부분 `/clarify`가 필요합니다
- "사용자가 다음에 뭘 해야 하는지"를 알려주는 방향으로 개선합니다
- `/harden`과 함께 쓰면 좋습니다 — 에러 처리 + 에러 메시지 개선 조합입니다

---

### /optimize [target] — 성능 개선

#### 무엇을 하는가

**성능을 개선**합니다.
로딩 속도, 렌더링 효율, 애니메이션 부드러움, 번들 사이즈를 최적화합니다.

#### 사용법

```
/optimize                  # 전체 성능 개선
/optimize images           # 이미지 최적화
/optimize animations       # 애니메이션 성능 개선
/optimize bundle           # 번들 사이즈 최적화
```

#### 꿀팁

- `/audit`에서 "성능" 점수가 낮으면 `/optimize`를 실행합니다
- `/animate` 또는 `/overdrive` 이후 성능이 걱정되면 `/optimize`로 보완합니다
- 불필요한 리렌더링, 과도한 DOM 조작, 미사용 CSS 제거에 특히 효과적입니다

---

### /harden [target] — 엣지 케이스 강화

#### 무엇을 하는가

**에러 처리, i18n, 텍스트 오버플로, 엣지 케이스**를 강화합니다.
정상 경로에서는 보이지 않지만 실제 사용에서 문제가 되는 부분을 잡습니다.

| 강화 항목       | 예시                                      |
| --------------- | ----------------------------------------- |
| 에러 처리       | 네트워크 실패, 빈 응답, 타임아웃          |
| i18n            | 긴 번역 텍스트, RTL 레이아웃              |
| 텍스트 오버플로 | 매우 긴 이름, 특수문자, 줄바꿈            |
| 엣지 케이스     | 빈 리스트, 단일 항목, 수천 개 항목        |

#### 사용법

```
/harden                    # 전체 강화
/harden checkout           # 결제 플로우 강화
/harden user-profile       # 프로필 관련 강화
```

#### 꿀팁

- 해피 패스만 구현했다면 반드시 `/harden`을 실행합니다
- `/critique` → `/harden` 조합으로 UX 리뷰 후 견고성을 보강합니다
- "사용자 이름이 200자면?" 같은 극단적 시나리오를 자동으로 테스트합니다

## 디자인 조정

### /animate [target] — 모션/애니메이션 추가

#### 무엇을 하는가

**의미 있는 애니메이션과 마이크로 인터랙션**을 추가합니다.
장식이 아닌, 사용자에게 피드백과 맥락을 전달하는 모션입니다.

#### 사용법

```
/animate                   # 전체에 적절한 모션 추가
/animate buttons           # 버튼 인터랙션 애니메이션
/animate page-transitions  # 페이지 전환 모션
```

#### 꿀팁

- `prefers-reduced-motion`을 자동으로 존중합니다
- 과하다 싶으면 `/quieter`로 톤다운할 수 있습니다
- `/animate` 후 `/optimize animations`로 성능을 확인하면 안전합니다

---

### /colorize [target] — 전략적 색상 추가

#### 무엇을 하는가

**단조로운 디자인에 전략적으로 색상을 추가**합니다.
무채색 위주의 UI에 포인트 컬러, 상태별 색상, 시각적 구분을 더합니다.

#### 사용법

```
/colorize                  # 전체에 색상 전략 적용
/colorize sidebar          # 사이드바 색상 개선
/colorize status-badges    # 상태 뱃지 색상 체계화
```

#### 꿀팁

- `.impeccable.md`의 브랜드 컬러를 기반으로 색상을 선택합니다
- 접근성(색 대비 4.5:1)을 자동으로 준수합니다
- `/colorize` 후 `/audit accessibility`로 색 대비를 확인하면 확실합니다

---

### /bolder [target] — 임팩트 강화

#### 무엇을 하는가

**밋밋한 디자인을 더 임팩트 있게** 만듭니다.
타이포그래피 강조, 색상 대비 증가, 시각적 무게감 추가로 존재감을 높입니다.

#### 사용법

```
/bolder                    # 전체 임팩트 강화
/bolder hero               # 히어로 섹션 강화
/bolder cta-buttons        # CTA 버튼 강조
```

#### 꿀팁

- `/critique`에서 "시각적 위계가 약하다"는 피드백이 나오면 `/bolder`를 사용합니다
- 과했다면 `/quieter`로 조절합니다 — 이 둘은 볼륨 조절 관계입니다
- 랜딩 페이지, CTA 영역에 특히 효과적입니다

---

### /quieter [target] — 차분하게 톤다운

#### 무엇을 하는가

**과한 디자인을 차분하고 세련되게** 만듭니다.
색상 채도 낮추기, 그림자 줄이기, 시각적 노이즈 제거에 집중합니다.

#### 사용법

```
/quieter                   # 전체 톤다운
/quieter alerts            # 알림 디자인 차분하게
/quieter background        # 배경 요소 톤다운
```

#### 꿀팁

- `/bolder`와 반대 방향입니다 — 둘을 번갈아 사용하며 적절한 톤을 찾습니다
- 대시보드처럼 장시간 보는 화면에는 `/quieter`가 특히 중요합니다
- 전문적이고 신뢰감 있는 느낌을 만들 때 유용합니다

---

### /delight [target] — 즐거운 터치 추가

#### 무엇을 하는가

**즐거움과 개성 있는 터치**를 추가합니다.
빈 상태 일러스트, 성공 애니메이션, 이스터에그, 유쾌한 마이크로카피 등입니다.

#### 사용법

```
/delight                   # 전체에 즐거운 터치 추가
/delight empty-states      # 빈 상태 디자인 개선
/delight success-screens   # 성공 화면 축하 효과
```

#### 꿀팁

- 기능이 완성된 후 마지막 단계에서 사용합니다
- "전문적이지만 친근한" 균형을 자동으로 맞춥니다
- `/delight` 후 과하다 싶으면 `/quieter`로 미세 조정합니다

---

### /overdrive [target] — 기술적 극한

#### 무엇을 하는가

**셰이더, 물리 기반 애니메이션, 스크롤 효과, 3D 변환** 등 기술적으로 극한까지 밀어붙입니다.
포트폴리오, 랜딩 페이지, 데모 등 강렬한 인상이 필요한 곳에 사용합니다.

#### 사용법

```
/overdrive                 # 전체에 고급 효과 적용
/overdrive hero            # 히어로 섹션에 극한 효과
/overdrive scroll          # 스크롤 기반 효과 추가
```

#### 꿀팁

- 일반 서비스보다는 **쇼케이스, 포트폴리오, 랜딩 페이지**에 적합합니다
- 성능 저하가 불가피하므로 `/optimize`를 반드시 이어서 실행합니다
- `prefers-reduced-motion` 사용자를 위한 폴백이 자동으로 포함됩니다

### /extract [target] — 컴포넌트/토큰 추출

#### 무엇을 하는가

반복되는 패턴을 **재사용 가능한 컴포넌트나 디자인 토큰으로 추출**합니다.
코드 중복을 줄이고 디자인 시스템의 일관성을 높입니다.

#### 사용법

```
/extract                   # 자동으로 추출 가능한 패턴 탐지
/extract card-patterns     # 카드 패턴을 컴포넌트로 추출
/extract colors            # 반복 색상을 디자인 토큰으로 추출
```

#### 꿀팁

- `/audit`에서 "안티패턴" 점수가 낮으면 추출이 필요하다는 의미입니다
- 추출 후 `/normalize`를 실행하면 새 컴포넌트가 시스템에 자연스럽게 통합됩니다
- 3번 이상 반복되는 패턴을 우선 추출 대상으로 잡습니다

### /adapt [target] [context] — 디바이스/화면 적응

#### 무엇을 하는가

다양한 디바이스와 화면 크기에 **적응하는 레이아웃**을 구현합니다.
브레이크포인트 설정, 터치 타겟 크기 조정, 화면별 레이아웃 변환을 처리합니다.

#### 사용법

```
/adapt dashboard mobile          # 대시보드를 모바일에 적응
/adapt sidebar tablet            # 사이드바를 태블릿에 적응
/adapt navigation small-screen   # 내비게이션을 작은 화면에 적응
```

#### 꿀팁

- `/audit`에서 "반응형" 점수가 낮으면 `/adapt`가 필요합니다
- `context`에 구체적인 디바이스를 지정하면 더 정확합니다 (예: `mobile`, `tablet`, `4k`)
- 터치 타겟 최소 44x44px을 자동으로 보장합니다

### /onboard [target] — 온보딩/첫 경험 설계

#### 무엇을 하는가

**온보딩 플로우, 빈 상태, 첫 실행 경험**을 설계합니다.
새 사용자가 처음 서비스를 만날 때의 경험을 체계적으로 구성합니다.

#### 사용법

```
/onboard                   # 전체 온보딩 플로우 설계
/onboard dashboard         # 대시보드 첫 경험 설계
/onboard empty-states      # 빈 상태 디자인 추가
```

#### 꿀팁

- 기능 구현 후 빈 상태를 방치하는 것은 흔한 실수입니다 — `/onboard`로 해결합니다
- 단계별 안내, 툴팁 투어, 샘플 데이터 등을 자동으로 설계합니다
- `/delight`와 조합하면 첫 인상이 훨씬 좋아집니다

### /typeset [target] — 타이포그래피 개선

#### 무엇을 하는가

**폰트 선택, 위계, 사이즈, 줄간격, 가독성**을 개선합니다.
텍스트가 많은 화면에서 읽기 편한 경험을 만듭니다.

#### 사용법

```
/typeset                   # 전체 타이포그래피 개선
/typeset blog              # 블로그 본문 가독성 개선
/typeset headings          # 제목 위계 재설정
```

#### 꿀팁

- 본문 텍스트의 이상적인 줄 길이(60~75자)를 자동으로 조정합니다
- `/bolder`와 다릅니다 — `/typeset`은 가독성, `/bolder`는 시각적 무게감입니다
- 다국어 지원이 필요하면 `/harden`과 함께 사용합니다

---

### /arrange [target] — 레이아웃/스페이싱 개선

#### 무엇을 하는가

**레이아웃 구조, 스페이싱 리듬, 시각적 정렬**을 개선합니다.
요소 간 간격, 그리드 정렬, 여백 일관성을 조정합니다.

#### 사용법

```
/arrange                   # 전체 레이아웃 개선
/arrange sidebar           # 사이드바 레이아웃 조정
/arrange card-grid         # 카드 그리드 스페이싱 개선
```

#### 꿀팁

- 8px 배수 그리드 시스템을 기본으로 적용합니다
- `/typeset`과 함께 사용하면 텍스트와 레이아웃 모두 정리됩니다
- "뭔가 어색한데 뭔지 모르겠다" 싶을 때 `/arrange`가 해결하는 경우가 많습니다

## 명령어 조합 워크플로우

### 전체 품질 개선 사이클

```
/audit blog                # 현재 상태 점수 확인
/normalize blog            # 디자인 시스템 표준 정렬
/polish blog               # 마이크로 디테일 수정
```

### UX 리뷰 + 견고성 강화

```
/critique checkout         # UX 디자인 리뷰
/harden checkout           # 에러 처리 + 엣지 케이스 강화
/clarify checkout          # UX 카피 개선
```

### 디자인 톤 조정

```
/bolder landing            # 임팩트 강화
/quieter dashboard         # 대시보드는 차분하게
/delight empty-states      # 빈 상태에 즐거운 터치
```

### 신규 페이지 완성도 높이기

```
/arrange new-page          # 레이아웃/스페이싱 잡기
/typeset new-page          # 타이포그래피 정리
/adapt new-page mobile     # 모바일 적응
/onboard new-page          # 첫 경험 설계
```

### 출시 전 최종 체크

```
/audit                     # 기술적 품질 점수 확인
/critique                  # UX 디자인 리뷰
/harden                    # 엣지 케이스 강화
/polish                    # 최종 마무리
```
