# DESIGN.md — AI에게 "이 사이트처럼 만들어줘"가 통하는 시대가 왔습니다

> Google Stitch가 만든 DESIGN.md 개념과, 58개 유명 사이트의 디자인 시스템을 모아둔 Awesome DESIGN.md 프로젝트를 소개합니다.

## 이런 경험 있으신가요?

AI에게 "대시보드 만들어줘"라고 하면 매번 비슷한 결과가 나옵니다. Inter 폰트, 보라색 그라데이션, 카드 안에 카드. 소위 **AI Slop**이라고 불리는 현상입니다.

"Vercel 느낌으로 만들어줘"라고 해도 AI는 Vercel이 정확히 어떤 색상을 쓰는지, 어떤 폰트를 쓰는지, 그림자를 어떻게 처리하는지 모릅니다. 기억에 의존한 **대충 비슷한** 결과가 나올 뿐입니다.

DESIGN.md는 이 문제를 해결합니다.

---

## DESIGN.md란 무엇인가

[Google Stitch](https://stitch.withgoogle.com/docs/design-md/overview/)에서 처음 도입한 개념입니다. **AI가 읽는 디자인 시스템 문서**입니다.

Figma 파일도 아니고, JSON 스키마도 아닙니다. 그냥 **마크다운 파일 하나**입니다. 프로젝트 루트에 놓으면 AI 코딩 도구가 자동으로 읽고, 그 디자인 시스템을 따라서 UI를 만듭니다.

### 왜 마크다운인가

LLM(대규모 언어 모델)이 가장 잘 이해하는 형식이 마크다운이기 때문입니다. 별도 파싱이나 변환 도구가 필요 없습니다. Claude Code, Cursor, Gemini CLI 등 **모든 AI 코딩 도구**가 바로 읽을 수 있습니다.

### 기존 방식과 비교

| 방식 | 장점 | 단점 |
|------|------|------|
| **Figma → 코드** | 시각적으로 정확함 | AI가 직접 읽지 못함, 변환 과정 필요 |
| **디자인 토큰 JSON** | 프로그래밍 친화적 | LLM이 맥락을 이해하기 어려움 |
| **프롬프트로 설명** | 간편함 | 매번 반복, 일관성 없음 |
| **DESIGN.md** | AI가 바로 읽음, 맥락 포함, 일관성 | 마크다운 작성 필요 |

핵심 차이는 DESIGN.md에는 **"왜 이렇게 해야 하는지"**가 포함되어 있다는 것입니다. 단순히 "색상: #171717"이 아니라 "순수 검정이 아닌 약간의 따뜻함이 있는 near-black으로, 눈의 피로를 줄이기 위해 사용합니다"처럼 의도와 맥락이 담겨 있습니다.

## DESIGN.md 안에는 무엇이 들어있는가

실제 Vercel의 DESIGN.md 일부를 살펴보겠습니다.

### 1. Visual Theme & Atmosphere

단순한 색상 코드 나열이 아닙니다. **디자인 철학**이 담겨 있습니다.

> "Vercel의 디자인은 개발자 인프라를 투명하게 만드는 시각적 논문입니다. 갤러리 같은 빈 공간에서 모든 요소가 자신의 픽셀을 정당화합니다. 이것은 장식으로서의 미니멀리즘이 아니라, 공학 원칙으로서의 미니멀리즘입니다."

AI가 이 맥락을 읽으면, 단순히 색상을 맞추는 것을 넘어 **디자인 의도**를 이해하고 코드를 생성합니다.

### 2. Color Palette & Roles

색상마다 **역할과 이유**가 명시되어 있습니다.

| 이름 | HEX | 역할 |
|------|-----|------|
| Vercel Black | `#171717` | 주 텍스트. 순수 검정이 아닌 약간의 따뜻함으로 가독성 향상 |
| Ship Red | `#ff5b4f` | 프로덕션 배포 워크플로우 단계 표시. 따뜻하고 긴급한 코랄 레드 |
| Preview Pink | `#de1d8d` | 프리뷰 배포 워크플로우. 비비드 마젠타 핑크 |
| Border Shadow | `rgba(0,0,0,0.08)` | 전통적 CSS border 대신 box-shadow로 구현한 보더 |

### 3. Typography Rules

폰트 사이즈뿐 아니라 **letter-spacing, line-height, weight, OpenType 기능**까지 전부 명시되어 있습니다.

Vercel의 경우 헤드라인에서 **-2.4px~-2.88px의 극단적 네거티브 레터 스페이싱**을 사용합니다. "코드를 프로덕션용으로 압축한 것처럼 텍스트를 압축한다"는 의도입니다. 이런 디테일은 프롬프트로 전달하기 거의 불가능하지만, DESIGN.md에는 자연스럽게 담깁니다.

### 4~9. 나머지 섹션

| 섹션 | 담고 있는 내용 |
|------|-------------|
| Component Stylings | 버튼, 카드, 인풋의 상태별 스타일 (hover, active, disabled) |
| Layout Principles | 스페이싱 스케일, 그리드, 여백 철학 |
| Depth & Elevation | 그림자 시스템 (Vercel은 multi-layer shadow stack 사용) |
| Do's and Don'ts | 이 디자인 시스템에서의 안티패턴 |
| Responsive Behavior | 브레이크포인트, 터치 타겟, 모바일 전략 |
| Agent Prompt Guide | AI에게 바로 쓸 수 있는 프롬프트 예시 |

## 사용법은 놀라울 정도로 간단합니다

### 전체 과정

```
1. 원하는 사이트의 DESIGN.md를 다운로드
2. 프로젝트 루트에 복사
3. AI에게 "이 DESIGN.md를 따라서 만들어줘"
```

이것이 전부입니다. 설치도 없고, 빌드도 없고, 설정 파일도 없습니다.

### 구체적 예시

Stripe 느낌의 결제 페이지를 만들고 싶다면:

```bash
# DESIGN.md 다운로드
curl -o DESIGN.md https://raw.githubusercontent.com/VoltAgent/awesome-design-md/main/design-md/stripe/DESIGN.md
```

그리고 AI에게:

```
이 DESIGN.md를 참고해서 결제 페이지를 만들어줘.
카드 입력 폼 + 주문 요약 + 결제 버튼 구성으로.
```

AI가 Stripe의 퍼플(`#533afd`), sohne-var 폰트, 블루 틴트 그림자, weight-300 헤드라인을 **자동으로 적용**해서 코드를 생성합니다.

## 각 사이트에 포함된 파일

DESIGN.md만 있는 것이 아닙니다.

| 파일 | 용도 |
|------|------|
| `DESIGN.md` | AI가 읽는 디자인 시스템 문서 |
| `preview.html` | 색상 스와치, 타이포 스케일, 버튼, 카드를 시각적으로 보여주는 카탈로그 |
| `preview-dark.html` | 다크모드 버전 카탈로그 |

`preview.html`을 브라우저에서 열면 해당 사이트의 디자인 시스템을 한눈에 확인할 수 있습니다. DESIGN.md를 선택하기 전에 미리보기로 활용하면 좋습니다.

## 마무리

DESIGN.md의 핵심은 **"AI에게 디자인 의도를 전달하는 가장 효율적인 방법"**이라는 것입니다.

프롬프트로 "Vercel처럼 만들어줘"라고 하면 AI의 기억에 의존한 **대략적인 결과**가 나옵니다. 하지만 DESIGN.md를 주면 색상 HEX 코드, 폰트 사이즈, 그림자 값, 심지어 letter-spacing까지 **정확한 결과**가 나옵니다.

58개 사이트의 디자인 시스템이 마크다운 파일 하나로 정리되어 있고, 복사+붙여넣기 한 번이면 적용됩니다. 이것이 DESIGN.md가 출시 1주일 만에 3만 Stars를 받은 이유입니다.
