# CLAUDE.md — 바이브 코딩 공통 설정

> 웹/앱 프로젝트 공통으로 쓸 수 있는 바이브 코딩 최적화 설정.
> CLAUDE-web.md 또는 CLAUDE-app.md와 함께 사용하거나 단독으로 사용.

---

## 바이브 코딩 원칙

### 빠르게 작동하는 것 먼저
- 완벽한 코드보다 **동작하는 코드 먼저**
- 리팩토링은 나중에 — 지금은 기능 구현에 집중
- "나중에 최적화"는 TODO 주석으로 마킹

### 작은 단계로
- 한 번에 하나의 기능만 구현
- 각 단계에서 동작 확인 후 다음으로
- 커밋은 자주, 작게

### 내가 원하는 것을 명확히
- 불명확한 요구사항이면 **구현 전에 질문**
- 여러 방법이 있으면 트레이드오프 설명 후 선택 요청
- 예상 동작을 설명하면 그에 맞게 구현

## 기술 스택

- **언어**: TypeScript (strict mode)
- **패키지 매니저**: pnpm

> 프로젝트별 추가 스택은 아래에 기록

---

## 명령어

```bash
# 실제 프로젝트 명령어로 교체
pnpm dev        # 개발 서버
pnpm build      # 빌드
pnpm typecheck  # 타입 체크
pnpm test       # 테스트
pnpm lint       # 린트
```

---

## 코딩 컨벤션

- TypeScript `any` 금지 — 모르면 `unknown`
- 함수형 컴포넌트만 사용
- 파일명: 컴포넌트 PascalCase, 유틸리티 camelCase
- import: ES modules (`import/export`), CommonJS 금지

## Git 규칙

- **커밋 메시지**: `feat: 기능`, `fix: 버그`, `chore: 작업`, `refactor: 리팩토링`
- **브랜치**: `feature/이름`, `bugfix/이름`

## 바이브 코딩 시 Claude에게 원하는 것

### 해줬으면 하는 것
- **구현 전 계획 한 줄 설명** ("이렇게 구현할게" 한 마디)
- **실행 가능한 코드** — 에러 없이 바로 돌아가는 코드
- **기존 패턴 유지** — 이미 있는 컴포넌트/함수 스타일 따르기
- **의존성 추가 시 알림** — 새 패키지 설치 전 알려줄 것

### 하지 않았으면 하는 것
- 요청하지 않은 리팩토링
- 과도한 추상화 (나중에 쓸 수도 있는 유틸리티 함수)
- 과도한 주석 (코드가 명확하면 주석 불필요)
- 에러 처리를 핑계로 기능 구현 미루기
