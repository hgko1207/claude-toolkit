# Claude Dispatch 완전 가이드 - 핸드폰으로 PC 작업 원격 지시하기

> Claude Dispatch는 핸드폰에서 명령하면 PC의 Claude가 알아서 작업하는 기능입니다. 효율적으로 쓰려면 Cowork 폴더 구조 세팅이 핵심입니다.

## 핵심 요약

- **Dispatch** = 핸드폰으로 명령 → PC에서 Claude가 자동 작업 (PC는 켜져 있어야 합니다)
- 잘 쓰려면 **Cowork 폴더 구조**(context, project, output, inbox, template)를 먼저 세팅해야 합니다
- 단순 채팅(질문-답변)이 아니라 **구조화된 자료 기반**으로 써야 제대로 된 결과가 나옵니다
- 현재 **베타 버전**이라 느리고 성공률이 낮지만, 방향성은 확실합니다

## Part 1: Dispatch란?

Claude Dispatch는 **핸드폰에서 업무를 지시하면 PC에서 Claude가 자동으로 작업을 수행**하는 기능입니다.

### 한 줄 비유

퇴근길 지하철에서 핸드폰으로 "보고서 만들어줘" → 다음날 출근하면 PC에 완성되어 있음

### 작동 원리

```
핸드폰 (Claude 앱)  ←→  PC (Claude 앱)
    명령 입력              자동 작업 수행
    결과 확인              파일 생성/저장
```

| 항목 | 내용 |
|------|------|
| 위치 | Claude 앱 → Cowork → 좌측 상단 **Dispatch** |
| 전제 조건 | PC가 **켜져 있어야** 합니다 |
| 연결 방법 | PC + 핸드폰 양쪽에서 Claude 앱 → Dispatch 모드 켜고 연결 |
| 현재 상태 | 베타 버전 |

---

## Part 2: 시작하기 — 연결 방법

### 설정 단계

1. **PC**: Claude 앱 다운로드 → 실행 → 좌측 상단 Dispatch 클릭
2. **핸드폰**: Claude 앱 → Cowork → 좌측 상단 Dispatch 클릭
3. **연결**: 양쪽에서 연결 버튼 누르면 자동 페어링

### 필수 체크사항

- PC **절전 모드 해제** — 컴퓨터가 꺼지면 Dispatch도 중단됩니다
- PC에서 **"모든 브라우저 작업 허용"** 켜기 — 웹 기반 작업 시 필요합니다
- 구글 드라이브 등 외부 서비스는 **커넥터로 연결**해두면 편리합니다

## Part 3: Cowork 폴더 구조 세팅

> Dispatch를 잘 쓰는 사람과 못 쓰는 사람의 차이는 **폴더 구조 세팅**에 있습니다.

### 왜 폴더 구조가 중요한가?

Dispatch를 단순 채팅(질문-답변)으로 쓰면 불만족스러운 결과가 나옵니다. Claude가 참조할 **자료와 맥락(Context)**이 없기 때문입니다. 폴더에 자료를 체계적으로 쌓아두면 Claude가 이를 기반으로 정확한 작업을 수행합니다.

### 추천 폴더 구조

```
claude-cowork/
├── context/              # 나의 정보, 스타일, 규칙
│   ├── about-me.md       # 기본 정보 (역할, 목표, 선호)
│   ├── voice-style.md    # 답변 톤, 포맷, 작성 규칙
│   └── rules.md          # 작업 규칙, 제약 조건
├── project/              # 진행 중인 프로젝트별 자료
├── output/               # AI가 만든 결과물 저장
├── inbox/                # 처리할 자료, 요청 대기
└── template/             # 반복 사용하는 템플릿
```

### 세팅 방법 (2가지)

#### 방법 1: 구조를 직접 전달

```
이 구조로 폴더 세팅해줘:

context/
  about-me.md
  voice-style.md
  rules.md
project/
output/
inbox/
template/
```

Claude Cowork 채팅에 위 구조를 붙여넣으면 자동으로 폴더와 파일을 생성합니다.

#### 방법 2: 완성된 결과물 역공학 (추천)

누군가 잘 만든 구조를 통째로 가져와서 세팅한 뒤, 내 상황에 맞게 조정하는 방식입니다. 원리를 먼저 파악하는 것보다 **이미 검증된 포맷에 나를 맞추는 게 더 빠릅니다**.

### 파일 관리 도구 추천: Anti-Gravity

폴더/파일 구조를 Claude 앱 안에서 보면 불편합니다. **Anti-Gravity** 같은 파일 뷰어를 쓰면 전체 구조를 한눈에 볼 수 있고, 마크다운 파일 내용도 바로 확인 가능합니다.

## Part 4: 글로벌 지침 설정

Cowork의 **모든 폴더에 공통 적용되는 규칙**을 설정할 수 있습니다.

### 설정 경로

```
프로필 → 설정 → 협업 → 작업 지침 → 편집
```

### 글로벌 지침에 넣을 내용

| 항목 | 예시 |
|------|------|
| 나는 누구인지 | 역할, 직업, 관심 분야 |
| 답변 포맷 | 마크다운, 표 우선, 간결하게 |
| 답변 톤 | 비즈니스, 친근함, 반말/존댓말 |
| 작업 규칙 | 항상 출처 포함, 파일로 저장 등 |
| 참조 경로 | context/ 폴더의 파일들 경로 명시 |

### CLAUDE.md와의 관계

글로벌 지침은 결국 **CLAUDE.md 역할**을 합니다. 폴더 구조가 변경되면 글로벌 지침도 함께 업데이트해야 합니다. Claude에게 "지금 수정된 구조에 맞게 글로벌 지침 수정해줘"라고 하면 자동으로 반영해줍니다.

## Part 5: 활용 사례

### 직장인

| 시나리오 | 명령 예시 |
|---------|----------|
| 퇴근길 보고서 준비 | "구글 드라이브에 있는 매출 데이터로 보고서 만들어줘" |
| 자료 가공 | "이 문서들 취합해서 다른 포맷으로 정리해줘" |
| 리서치 | "최근 AI 뉴스 5개 조사해서 보고서 파일로 만들어줘" |
| PPT 초안 | "카페 매출 데이터 분석해서 구글 슬라이드 7장으로 만들어줘" |

### 크리에이터

| 시나리오 | 명령 예시 |
|---------|----------|
| 콘텐츠 기획 | "요즘 트렌드 키워드 조사해줘" |
| 대본 작성 | "이 주제로 유튜브 대본 초안 작성해줘" |
| 썸네일 카피 | "이 영상 제목으로 카피 5개 만들어줘" |

### 자영업자 / 사업자

| 시나리오 | 명령 예시 |
|---------|----------|
| 매출 분석 | "이 데이터 기반으로 인사이트 분석 보고서 만들어줘" |
| 마케팅 전략 | "다음 달 마케팅 전략 짜줘, 기존 자료 참고해서" |

### 핵심 포인트

> 자료와 데이터 근거가 없으면 AI는 감에 의해서 작업할 수밖에 없습니다. **자료를 잘 정리해서 폴더에 세팅**해두는 것이 가장 중요합니다.

## Part 6: 장단점 정리

### 장점

| 장점 | 설명 |
|------|------|
| **공간 제약 없음** | 지하철, 버스, 택시 어디서든 핸드폰으로 작업 지시 가능 |
| **보안 우수** | 내 PC에서만 작업, 외부 서버 업로드 불필요 |
| **대화 기록 유지** | 과거 세션이 기록되어 점점 정확한 결과물 생성 |

### 단점

| 단점 | 설명 |
|------|------|
| **속도 느림** | 작업 완료까지 시간이 오래 걸림 |
| **성공률 낮음** | 베타 버전이라 불안정, 디테일한 부분에서 아쉬움 |
| **PC 상시 가동 필요** | 절전 모드 해제 + 컴퓨터 항상 켜둬야 함 |

### 대안: Discord/Telegram 연동

PC가 꺼져도 작업이 돌아가게 하려면 Discord나 Telegram에 AI 에이전트를 연결하는 방법이 있습니다. 다만 세팅 난이도가 매우 높아서 (약 1주일 소요) 비개발자에게는 어려울 수 있습니다. Dispatch는 이에 대한 **쉬운 대안**입니다.

---

## 마무리

Claude Dispatch는 아직 100% 실무 투입 수준은 아니지만, **AI 에이전트가 시공간 제약 없이 일하는 시대**의 입문 단계입니다.

지금부터 준비할 것:
1. **Cowork 폴더 구조** 세팅 — context, project, output, inbox, template
2. **글로벌 지침** 작성 — 나의 정보, 규칙, 참조 경로
3. **자료 축적** — AI가 참고할 데이터를 꾸준히 정리

도구의 성능이 올라가는 건 시간 문제입니다. 지금 **구조와 자료를 잘 세팅해둔 사람**이 안정화됐을 때 가장 먼저 혜택을 봅니다.
