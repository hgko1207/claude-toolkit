## 입문

### 1. CLAUDE.md — 프로그레시브 디스클로저

모든 규칙을 CLAUDE.md에 다 적지 말고, **어디서 찾을 수 있는지만** 적으세요.

```markdown
# 코딩 컨벤션

→ docs/conventions.md 참고

# API 설계 원칙

→ docs/api-guide.md 참고
```

신입한테 매뉴얼 전체를 주는 것보다 목차만 주는 게 낫습니다.

### 2. 시스템 프롬프트 다이어트 — MCP 관리

MCP/플러그인을 많이 설치해두면 대화 시작 시 읽어야 할 컨텍스트가 ~2만 토큰까지 불어납니다.

| 상태           | 토큰                 |
| -------------- | -------------------- |
| MCP 다 켜둠    | ~20만 → 7만으로 감소 |
| 필요한 것만 켬 | ~9,000               |

**실전 방법:** MCP는 많이 설치해도 되지만, 동시에 켜두는 건 5~6개로 제한하세요.

### 3. 상태 줄 (Status Line)

현재 토큰 사용량을 실시간으로 표시합니다. 눈에 보여야 관리가 됩니다.

```
/status
```

## 중급

### 4. 컨텍스트는 우유입니다

대화가 길어질수록 앞 내용이 흐릿해집니다. 자동 압축에 맡기지 말고 적절한 타이밍에 직접 정리하세요.

```
/compact
```

**정리 타이밍:**

- 큰 기능 하나를 완성했을 때
- 작업 방향이 바뀔 때

### 5. 모델 선택법

| 작업                        | 모델   |
| --------------------------- | ------ |
| 파일 찾기, 간단한 수정      | Haiku  |
| 여러 파일 동시 수정, 코딩   | Sonnet |
| 전체 구조 설계, 복잡한 버그 | Opus   |

모든 작업에 Opus를 쓸 필요는 없습니다. 간단한 식사에 코스 요리를 시키지 않는 것과 같습니다.

### 6. 플랜 모드 먼저

코딩부터 시작하면 잘못된 방향으로 달려가 토큰만 낭비합니다. 계획서를 먼저 작성하게 하고, 검토 후 OK하면 코딩을 시작하세요.

```
계획만 세워줘, 실행하지 마
```

설계도 없이 벽돌부터 쌓지 않습니다.

### 7. 레퍼런스 코드 함께 제공

비슷한 오픈소스 코드를 같이 보여주면 결과물 품질이 확 달라집니다.

```
이 코드를 참고해서 만들어줘: [github 링크 or 코드 붙여넣기]
```

빈 종이에 그리는 것과 참고작품 보고 그리는 건 완전히 다릅니다.

## 고급

### 8. 서브 에이전트

클로드 혼자 일하는 게 아니라 **팀을 꾸립니다**. 역할을 나눠 전문 에이전트에게 위임합니다.

Arpan의 시스템 예시 (에이전트 16개):

- 계획 에이전트
- 설계 에이전트
- 코딩 에이전트
- 코드 리뷰 에이전트

각 에이전트에게 할 일 하나만 주고, 결과물을 다음 에이전트에게 넘기는 파이프라인입니다.

→ 이 레포의 `@planner → @implementer → @reviewer → @deployer` 워크플로우가 이 개념입니다.

### 9. Git Worktree

워크트리를 쓰면 같은 프로젝트에서 **여러 작업 폴더를 동시에** 만들 수 있습니다.

```bash
git worktree add ../project-feature-a feature-a
git worktree add ../project-feature-b feature-b
```

각 폴더에서 Claude를 따로 실행하면 에이전트 5개가 동시에 다른 기능을 개발할 수 있습니다.

책상 1개로 일하던 걸 책상 5개 놓고 동시에 진행하는 것입니다.

### 10. 훅 (Hooks)

특정 시점에 자동으로 실행되는 명령어입니다. 알람 시계와 같습니다.

| 훅              | 실행 시점       | 용도                |
| --------------- | --------------- | ------------------- |
| `session-start` | 새 대화 시작 시 | 지난 기록 자동 로드 |
| `pre-compact`   | 메모리 정리 전  | 중요 내용 먼저 저장 |
| `stop`          | 대화 종료 시    | 배운 것 자동 기록   |

이 세 개를 조합하면 **대화가 끝나도 배운 걸 기억하는 시스템**이 완성됩니다.

## 주의사항

- MCP 너무 많이 켜두지 마세요 → 컨텍스트 윈도우가 감소합니다
- 자동 압축만 믿지 마세요 → 중요 맥락이 소실될 수 있습니다
- 프롬프트 인젝션에 주의하세요 → 외부 데이터 읽을 때 악의적 명령이 포함될 수 있습니다
