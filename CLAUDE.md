# Claude Toolkit 프로젝트

## 프로젝트 목적
Claude, Claude Code, Cowork 등 **Claude 관련 정보**를 유튜브, 블로그 등 다양한 소스에서 수집하여 체계적으로 정리하는 저장소.
티스토리 블로그 발행을 고려하여, 사람들이 읽기 쉽고 정보를 빠르게 습득할 수 있는 형태로 작성한다.

## 콘텐츠 범위
- **Claude 모델**: Claude Opus, Sonnet, Haiku 등 모델 분석, 비교, 업데이트
- **Claude Code**: CLI 도구, 에이전트, 스킬, MCP, 워크플로우 활용법
- **Claude 생태계**: API, SDK, Anthropic Console, Cowork, Claude Desktop 등
- **Claude 활용**: 프롬프팅, 실전 팁, 사례, 베스트 프랙티스
- 범위 밖의 정보(다른 AI 모델, 일반 AI 개념 등)는 다루지 않는다

## 폴더 구조 및 분류 기준
| 폴더 | 용도 | 예시 |
|------|------|------|
| `tips/` | 실전 꿀팁, 심층 가이드, 설정법 | 프롬프팅 팁, Hooks 가이드, 최적화 |
| `tips/templates/` | CLAUDE.md 바이브 코딩 템플릿 | 웹 앱, 모바일 앱, 공통 설정 |
| `gstack/` | Garry Tan의 가상 엔지니어링 팀 가이드 | 설치, 기획, 개발, QA, 배포 |
| `.claude/skills/` | 재사용 가능한 스킬 모음 | 스킬 생성, 프로젝트 초기화 |
| `.claude/agents/` | 서브 에이전트 정의 | planner, implementer, reviewer |

> 기존 폴더에 맞지 않는 새로운 주제는 적절한 폴더를 생성한다.

## 소스 분석 프로세스
사용자가 정보(유튜브, 블로그 등)를 제공하면:
1. **분석**: 내용을 깊이 이해하고 핵심을 추출
2. **검증**: 사실 관계 확인, 누락된 맥락 보완
3. **심화**: 단순 요약이 아닌, 배경 지식과 실전 인사이트를 추가
4. **분류**: 기존 폴더 구조에서 적절한 위치 결정. 맞는 폴더가 없으면 새 폴더 생성
5. **작성**: 아래 템플릿과 원칙에 따라 마크다운 작성
6. **저장**: 해당 폴더에 파일 생성

## 글 작성 규칙

### 파일명
- 소문자 영문, 하이픈(`-`) 구분: `claude-code-hooks-tips.md`
- 날짜 접두사 선택적: `2026-03-28-claude-4-analysis.md`

### 글 유형별 구조

#### 1. 팁 모음형 (예: pro-tips.md)
```markdown
# 제목

> 한 줄 요약

## 카테고리 A
### 팁 1: 제목
**문제**: 어떤 상황에서 불편한지
**해결**: 구체적인 방법 (코드블록 포함)
**핵심**: 한 줄 정리

### 팁 2: 제목
...
```

#### 2. 심층 가이드형 (예: hooks-guide.md, workflow-guide.md)
```markdown
# 제목

> 한 줄 요약

## Part 1: 개념/기초
### 개념 설명
| 항목 | 설명 | 비고 |
|------|------|------|

## Part 2: 실전 활용
### 레시피/패턴 1
(코드블록 + 설명)

## Part 3: 고급 (선택)
...
```

#### 3. 레퍼런스형 (예: cli-reference.md)
```markdown
# 제목

> 한 줄 요약

## 카테고리 A
| 명령어/항목 | 설명 | 예시 |
|-------------|------|------|

## 카테고리 B
(코드블록 중심)
```

### 공통 요소

| 항목 | 필수/선택 | 설명 |
|------|:---------:|------|
| H1 제목 | **필수** | 핵심을 한 줄로 |
| 한 줄 요약 (blockquote) | **필수** | `>` 로 시작하는 한 문장 요약 |
| 출처 | **필수** | 글 하단에 원본 링크 |
| 테이블 | **필수** | 비교/정리 시 적극 활용 |
| 코드블록 | **필수** | 설정/명령어는 반드시 코드블록으로 |
| 목차 | 선택 | 본문이 긴 글(H2 5개 이상)일 경우 |
| 핵심 요약 | 선택 | 3~5줄 핵심 포인트, 긴 글에 권장 |
| 작성일 | 선택 | `**작성일**: YYYY-MM-DD` |
| 태그 | 선택 | `**태그**: #ClaudeCode #팁` — 티스토리 발행 시 활용 |

### 문체
- **합쇼체**(~입니다, ~합니다, ~있습니다)를 사용한다
- 모든 글, 가이드, 설명에 통일 적용
- 해체(~이다), 해요체(~해요) 사용 금지

### 작성 원칙
1. **핵심 먼저**: 결론/요약을 글 상단에 배치 (역피라미드 구조)
2. **쉬운 말**: 전문 용어는 처음 나올 때 간단히 설명
3. **구조화**: 글머리 기호, 표, 번호 목록을 적극 활용
4. **실용적**: "그래서 뭘 할 수 있는데?"에 답하는 내용 포함
5. **적절한 길이**: 핵심에 집중, 불필요한 장황한 설명 제거
6. **출처 명시**: 원본 소스 링크 항상 포함

### 티스토리 블로그 친화적 작성
- **제목**: 검색되기 쉬운 명확한 제목 (예: "Claude Code 실전 활용법 - 프로젝트 세팅부터 배포까지")
- **소제목(H2, H3)**: 스캔하며 읽을 수 있도록 의미 있는 소제목 배치
- **단락 길이**: 한 단락 3~4문장 이내, 모바일에서도 읽기 편하게
- **이미지/코드블록**: 글 중간에 시각적 요소 배치하여 가독성 향상
- **태그**: 티스토리 태그로 바로 쓸 수 있게 #태그 형태로 하단에 포함
